DeerFlow 中文上手清单

        
      

      

        
适用范围：深度调研、任务编排、文件产出、线程记忆、工具调用、报告与网页生成。

        
能力总览：深度调研（网页检索/抓取/总结）、多步任务编排、生成 md/html/代码、线程内上下文记忆、调用搜索/抓取/文件读写/bash。

        
推荐操作路线：先定义目标与交付物 → 再给约束（时间、来源、格式）→ 上传材料 → 要求结构化输出 → 让它继续迭代补差。

        
文件上传后怎么分析：说明文件用途、要抽取的信息、输出格式；例如“读取上传的财报 PDF，抽取收入/毛利/费用/管理层指引，输出中文要点+风险表+3页汇报结构”。

        
如何让它生成网页/报告/代码：明确目标文件、目录、技术栈、验收标准；例如“生成单文件 HTML 仪表盘，包含图表占位、结论区、下载按钮，并输出到 runtime_data/artifacts/”。

        
可直接复制的提示词

        
1. 请对某行业做深度调研：检索近12个月一手资料，按市场规模、竞争格局、价格带、关键玩家、风险点输出中文报告，并附来源链接。

2. 基于上传的 Excel 和 PDF，先识别核心指标口径，再拆成数据清洗、异常检查、趋势分析、结论汇报四个子任务，逐步执行并输出结果文件。

3. 读取这份数据库 schema，生成可执行 SQL 分析方案：先列候选指标和维度，再给 SQL、结果解释、图表建议、PPT 页面结构。

4. 生成一个客户汇报网页：含封面、核心发现、图表区、建议动作、附录来源，输出 html/css/js 文件并说明如何本地预览。

5. 对这个代码仓做技术调研：说明模块职责、启动方式、关键依赖、风险点，并生成一份给新同学的中文上手文档。

        
高质量任务写法：目标 + 输入材料 + 分析维度 + 输出格式 + 约束条件 + 是否需要继续迭代。

      

    
    
      

        
角色 role 怎么用

        
      

      

        
role 决定这个字段在分析里扮演什么角色，不是备注，而是会影响后续查询计划。

        
metric 指标：拿来算数值。例：价格、销售额、销量。

        
dimension 维度：拿来分组。例：品牌、类目、颜色。

        
time 时间：拿来看趋势。例：上新时间、创建日期。

        
filter 过滤：拿来限制范围。例：性别、场景、是否新品。

        
最简单的记法：

        
按品牌看平均价格
品牌 -> dimension
价格 -> metric

看最近30天上新趋势
上新时间 -> time
价格/商品数 -> metric

只看女装里的品牌分布
性别 -> filter
品牌 -> dimension
