# 远程开发环境操作手册（Mac + Windows + WSL + Codex）
版本：V1.0  
日期：2026-03-08

## 1. 目标
本手册用于把你的开发工作从 Mac 本机迁移到 Windows 远程主机，Mac 仅作为轻量终端，降低内存和多窗口压力。

---

## 2. 架构说明
1. Windows 电脑作为远程主机，长期在线。
2. Windows 内运行 WSL2 Ubuntu，作为统一 Linux 开发环境。
3. 代码、依赖、构建、Codex 都在 WSL 内执行。
4. Mac 通过 SSH 连接 Windows，并进入 WSL 开发。
5. 通过 `tmux` 维持多项目会话，断网不丢进程。

---

## 3. 一次性准备

## 3.1 Windows 管理员 PowerShell
执行以下命令启用 SSH 服务：

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Start-Service sshd
Set-Service -Name sshd -StartupType Automatic
New-NetFirewallRule -Name sshd -DisplayName "OpenSSH Server" -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22
```

若未安装 WSL2：

```powershell
wsl --install -d Ubuntu
```

重启后首次进入 Ubuntu，创建 Linux 用户（建议：`dev`）。

## 3.2 WSL（Ubuntu）初始化
执行：

```bash
sudo apt update
sudo apt install -y git tmux curl unzip build-essential
mkdir -p ~/workspace
```

可直接使用脚本（在本项目中）：
- [wsl_setup.sh](/Users/chen/Desktop/Cursor_project/ai_money/fz_workflow/scripts/wsl_setup.sh)

## 3.3 WSL 安装 Codex
按你现有的 Codex 安装方式执行，验证命令：

```bash
codex --help
```

```bash

./scripts/setup_mac_remote_dev.sh \
  --host <windows_ip_or_tailscale_ip> \
  --user <windows_user> \
  --alias devwin \
  --wsl-user dev \
  --distro Ubuntu
```

## 4.2 分发公钥到 Windows

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub <windows_user>@<windows_ip_or_tailscale_ip>
```

```bash

```

## 5. 日常开发流程

## 5.1 进入远程 WSL

```bash
cd /Users/chen/Desktop/Cursor_project/ai_money/fz_workflow
./scripts/connect_wsl_remote.sh devwin dev Ubuntu
```

## 5.2 新项目初始化

```bash
cd ~/workspace
git clone <repo-url> proj-a
cd proj-a
tmux new -s proj-a
codex
```

## 5.3 恢复项目会话

```bash
tmux ls
tmux attach -t proj-a
```

## 5.4 多项目并行建议
1. 一个项目一个 `tmux session`，例如 `proj-a`、`proj-b`。
2. 每个项目独立依赖环境（`.venv` 或 `node_modules`）。
3. 所有构建、测试、索引都在 WSL 跑，不在 Mac 本机跑。

## 7. 常见问题排查

## 7.1 Mac 连接不上 Windows
1. Windows 端检查 `sshd`：

```powershell
Get-Service sshd
```

2. 检查 22 端口：

```powershell
netstat -ano | findstr :22
```

3. Mac 侧调试 SSH：

```bash
ssh -v devwin
```

## 7.2 能 SSH 但不能进入 WSL
1. Windows 先确认发行版名称：

```powershell
wsl -l -v
```

2. 按实际发行版替换连接命令中的 `Ubuntu`。

## 7.3 断线后任务中断
1. 必须在 `tmux` 内启动 Codex 和构建任务。
2. 使用如下恢复：

```bash
tmux attach -t <session-name>
```

## 7.4 WSL 性能差
1. 确保代码在 WSL 文件系统（`/home/dev/workspace`），不要放 `/mnt/c/...` 做重 IO 开发。
2. 关闭不必要的本机应用和后台服务。
3. 尽量分时运行大任务，避免多个大编译同时执行。

---

## 8. 上线前检查清单
1. Windows `sshd` 服务已启动并开机自启。
2. WSL 可正常进入，`git/tmux/codex` 命令可用。
3. Mac 能通过 `ssh devwin` 连接。
4. `connect_wsl_remote.sh` 可直接进入 WSL。
5. 至少 1 个项目已在远程跑通 `codex + 构建 + 测试`。
