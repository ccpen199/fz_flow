## 2. 业务目标与边界

### 2.1 目标
- 将面料开发流程从被动接单升级为主动企划（ODM 模式）。
- 在 5 分钟内自动生成《面料定向开发与企划方案》。
- 同时输出：风险预警、成本测算、差异化卖点、业务逼单话术。

### 2.2 本期范围
- 支持中文输入的非结构化业务需求解析。
- 支持多 Agent 并行分析与统筹汇总。
- 支持外部市场数据与内部历史数据联合推理。
- 支持结构化报告导出（JSON + Markdown + HTML）。

### 2.3 暂不包含
- 自动派发打样单给供应商（Phase 3 处理）。
- 全自动 BOM 与财务系统写回（先只读、后写入）。

---

## 3. 角色与核心场景

### 3.1 用户角色
- 业务员：输入客户需求，查看企划结果，输出沟通话术。
- 面料研发：查看工艺建议与风险红线，做工艺确认。
- 管理层：查看方案成功率、毛利预测、开发效率指标。

### 3.2 核心场景
- 场景 A：客户提出模糊目标（如“Lululemon 裸感 + 抗起球 + 环保 + 预算上限”）。
- 场景 B：老客户复购场景，需结合历史客诉自动规避踩坑方案。
- 场景 C：新品企划场景，需结合竞品差评 VOC 挖掘卖点。

---

## 4. 总体技术架构

### 4.1 架构风格
- 编排层：`Orchestrator + 5 Expert Agents`。
- 知识层：`SQL + RAG 双引擎`。
- 执行层：`工具调用 + 规则引擎 + 成本计算引擎`。
- 接入层：`Web/API + ERP/PLM + 爬虫数据管道`。

### 4.2 逻辑分层
1. `Interaction Layer`：业务员输入、报告展示、审批反馈。
2. `Agent Orchestration Layer`：任务拆解、并行调度、结果融合。
3. `Knowledge & Data Layer`：内部结构化数据 + 外部非结构化向量检索。
4. `Rule & Calculation Layer`：风险否决、成本测算、可生产性校验。
5. `Observability Layer`：日志、追踪、评估、告警。

### 4.3 推荐技术栈（可替换）
- 编排：LangGraph（代码型）或 Dify Workflow（低代码型）。
- API：FastAPI。
- 结构化存储：PostgreSQL。
- 向量库：Qdrant（MVP）或 Milvus（规模化）。
- 缓存/队列：Redis。
- 对象存储：S3 兼容存储（报告、原始抓取数据）。
- 模型接入：OpenAI/通义/DeepSeek（通过统一模型适配层）。

## 5. Multi-Agent 设计

### 5.1 Agent 清单
1. `Orchestrator`（统筹调度）
2. `Requirement Parser Agent`（需求解析专家）
3. `Internal History Agent`（内部历史侦探）
4. `Market Intelligence Agent`（品牌情报大脑）
5. `Material & Yarn Agent`（材质纱线专家）
6. `Color & Finishing Agent`（色彩染整专家）

### 5.2 Agent 输入输出契约（统一 JSON）
```json
{
  "task_id": "string",
  "agent_name": "string",
  "input": {},
  "output": {},
  "confidence": 0.0,
  "evidence": [],
  "risk_flags": [],
  "cost_impact": {}
}
```

### 5.3 Orchestrator 关键职责
- 意图识别与任务拆解。
- 并行调用专家 Agent（超时控制与重试）。
- 冲突消解（例如：功能可实现但成本超预算）。
- 汇总生成最终方案和可执行建议。

## 6. 数据底座设计

### 6.1 结构化数据（SQL）
- 客户主数据：`customer_profile`
- 历史订单：`sales_orders`
- 客诉与质量记录：`quality_claims`
- 纱线与原料价格：`yarn_price_daily`
- 染整工艺字典：`finishing_process_dict`

### 6.2 非结构化数据（RAG）
- 外部：竞品详情页、评论文本、价格监测快照。
- 内部：打样工艺单 PDF、SGS 测试报告、研发笔记。

### 6.3 索引策略
- 向量索引：语义检索（产品卖点、工艺经验、客诉原因）。
- 关键词索引：精确字段命中（成分比例、克重、测试等级）。
- 混合召回：`BM25 + Vector + Rerank`。

### 6.4 Chunk 切分建议
- 商品详情：每块 300-500 tokens，保留属性表整块。
- 用户评论：按单条评论切分，附评分、SKU、时间标签。
- 工艺文档：按“工序段”切分，保留温度/时长/助剂参数。

## 8. 工作流（端到端）

### 8.1 主流程
1. 业务员提交需求（自然语言）。
2. Orchestrator 调用需求解析 Agent，生成标准参数。
3. 并行触发 Agent 2/3/4/5 获取内部、外部、工艺建议。
4. 规则引擎执行一票否决与风险分级。
5. 成本引擎计算价格区间与超预算幅度。
6. Orchestrator 合成报告并生成话术建议。
7. 结果写入方案库，前端展示并支持人工反馈。

### 8.2 异常处理
- 数据缺失：降级为“弱证据模式”，报告中标注置信度。
- 超时：单 Agent 超时后返回 fallback 输出，不阻塞整体。
- 规则冲突：由 Orchestrator 选择“保守可生产方案”优先。

---

## 9. 规则引擎设计

### 9.1 风险规则类型
- `硬规则（Hard Stop）`：直接否决（如目标测试等级无法满足）。
- `软规则（Soft Warning）`：允许通过但强提醒（成本超预算、交期风险）。

### 9.2 示例规则
- `IF` 高弹力 + 荧光色 + 高日晒牢度要求 `THEN` hard_stop。
- `IF` 磨毛深度 > 阈值 且抗起球等级 >= 4  `THEN` hard_stop。
- `IF` 预计成本 > 预算 10% `THEN` soft_warning + 输出议价建议。

## 10. API 设计（MVP）

### 10.1 创建任务
- `POST /api/v1/proposal/tasks`
- 入参：自然语言需求 + 客户标识
- 出参：`task_id`

### 10.2 查询任务状态
- `GET /api/v1/proposal/tasks/{task_id}`
- 返回：`pending/running/success/failed` + 进度百分比

### 10.3 获取最终方案
- `GET /api/v1/proposal/tasks/{task_id}/result`
- 返回：结构化报告 + Markdown 报告 + 风险清单

### 10.4 反馈闭环
- `POST /api/v1/proposal/tasks/{task_id}/feedback`
- 用途：记录业务员采纳情况，反哺提示词和规则权重。

## 11. Prompt 与知识策略

### 11.1 Prompt 管理
- 按 Agent 单独维护 system prompt。
- 所有 prompt 版本化（`prompt_version` 字段入库）。
- 每次生成记录 `evidence_id`，保证可追溯。

### 11.2 证据引用规范
- 每条关键结论必须绑定证据来源。
- 若证据冲突，输出“冲突说明 + 建议人工复核项”。

## 12. 评估与监控

### 12.1 业务指标
- 报告生成时长 P95 < 5 分钟。
- 方案采纳率（业务员采用进入打样）> 40%（MVP 目标）。
- 客诉复发率（同类问题）下降 > 20%（季度目标）。

### 12.2 技术指标
- Agent 调用成功率 > 99%。
- RAG 命中可用证据率 > 85%。
- 成本估算误差（与实际打样核价）< 10%。

### 12.3 可观测性
- 全链路 trace（任务级、Agent 级、工具调用级）。
- 关键日志脱敏（客户名、价格、联系方式）。
- 告警：超时、空结果、规则冲突异常。

## 13. 分阶段实施计划

### Phase 1（2-3 周，MVP）
- 范围：Orchestrator + 需求解析 + 品牌情报 + 基础报告输出。
- 数据：仅接入外部抓取数据（CSV/Excel + RAG）。
- 交付：
  - 可对话输入需求并输出报告。
  - 支持基本预算提醒与卖点建议。
  - 支持人工下载报告（MD/HTML）。

### Phase 2（1-2 个月）
- 范围：加入内部历史侦探 + 材质/色彩专家 + 规则引擎。
- 数据：接入 ERP 只读 API，接入工艺字典与纱线库。
- 交付：
  - 客诉排雷能力上线。
  - 成本估算从规则估算升级为参数化估算。
  - 输出“可生产性评分”。

### Phase 3（3 个月+）
- 范围：全闭环自动化（核价/BOM/打样派单）。
- 数据：接入供应商协同和消息渠道（企微/邮件）。
- 交付：
  - 自动生成打样任务单。
  - 自动触发供应商通知与回执追踪。
  - 形成企划到打样的闭环数据资产。

## 15. 安全与权限

- ERP/PLM 默认只读接入，采用最小权限账号。
- 关键数据分级：公开、内部、敏感（价格/客户信息）。
- 对外模型调用前做脱敏（客户名、联系方式、合同价）。
- 审计要求：所有方案生成均记录“输入、证据、输出、版本”。

---

## 16. 验收标准（MVP）

满足以下即通过 MVP 验收：
- 输入 1 条自然语言需求，可在 5 分钟内生成完整报告。
- 报告包含：需求结构化、竞品洞察、建议配方、风险提示、预算判断、业务话术。
- 每条核心结论至少有 1 条可追溯证据。
- 失败任务可定位到具体 Agent 或数据源问题。
